# SportsYeti Form Controls — `@sports-yeti/ui`

> Reference for the unified form-control library shared by `apps/sports-yeti` (Expo iOS/Android) and `apps/sports-yeti-admin` (Expo Web). Generated by `@ux-architect` + `@ui-designer` + `@accessibility-auditor` + `@mobile-app-builder` + `@frontend-developer`. Lives at `docs/design-system-form-controls.md`. The mobile-only audit at `apps/sports-yeti/docs/design-system.md` is still the source of truth for screen-by-screen migration; this doc covers the shared form layer that powers both apps.

---

## Why a shared package?

Before this pass:

- `apps/sports-yeti/src/ui/Input.tsx` and `apps/sports-yeti-admin/src/ui/Input.tsx` were **two near-identical implementations** that drifted on density, addon support, and focus styling.
- `Switch`/`TextInput`/`<input type="file">`/free-text date entry was repeated **across 13+ screens** with copy-pasted token references.
- `Select` only existed on admin. `BottomSheet`-style picker only existed on mobile. Combobox, RadioGroup, Checkbox(Group), DatePicker, FileUpload didn't exist at all.

The new package consolidates this into one source of truth so the next mock-data refactor wave doesn't have to reinvent any field.

### Theme contract

Each app's `src/theme/ui-theme.ts` exports a `uiTheme: Theme` that maps that app's existing token files into the shape `@sports-yeti/ui` expects. Both apps wrap their root in `<UIThemeProvider value={uiTheme}>` so every primitive can call `useTheme()` to resolve colors, spacing, radii, shadows, and typography.

| Token slice | Mobile | Admin |
|---|---|---|
| `colors.brand.*` | identical (#006495 family) | identical (#006495 family) |
| `colors.status.*` | **upgraded** to AA-compliant darker variants (#2E7D32 / #B26200 / #C62828) inside the adapter | already AA-compliant (no change) |
| `colors.text.muted` | `#6B7785` (already fixed in mobile theme) | `#6B7785` |
| `spacing` | adds `'2xs': 2` inside the adapter to satisfy the contract | already exports `'2xs': 2` |
| `radii.lg` | `20` | `14` (admin tighter) |
| `shadows.popover` | reuses `shadows.card` (mobile only has card/soft/nav) | dedicated `shadows.popover` |
| `density` | `'comfortable'` (44pt min targets) | `'compact'` (32-44px) |
| `typography` | `PlusJakartaSans` named families × explicit weight | CSS family string + `fontWeight` numeric |

The contract intentionally narrows each app's full token set to a stable cross-platform subset. Local apps remain free to extend their own theme files for screen-only tokens that don't need to be portable (e.g., admin's `glassOverlay`, `sidebar`, `alpine` tokens).

### Density

Components consume `useTheme().density` and fork their internal sizing via `useControlDims()`. A single `<Input size="md">` is 38px tall on admin and 52pt tall on mobile; the same component, same prop, two ergonomically-correct results.

## The 18 primitives

| Primitive | Purpose | Replaces |
|---|---|---|
| `<Form>` | RHF FormProvider + zod resolver, defaults `mode='onBlur'` | per-screen `useState` + manual validation |
| `<FormField>` | label / description / error / required marker / FormFieldContext | every screen rolling its own label + error markup |
| `<FormSection>` | card-styled vertical stack with title/description | `<Card style={section}>` per screen |
| `<FormRow>` | responsive multi-column row (collapses below `breakpoint`) | `flex: 1, minWidth: 200` boilerplate |
| `<FormActions>` | submit + cancel pair, sticky on mobile, inline on admin | per-screen footer styling |
| `useFieldController()` | flat RHF adapter — value/onChange/onBlur/error in one shot | importing RHF types into every screen |
| `<Input>` | text/email/phone/url/search · `leftAddon`/`rightAddon`/`clearable`/`showCounter` · sm/md/lg | the two `Input.tsx` files + every raw `TextInput` |
| `<PasswordInput>` | eye toggle, optional strength meter, autofill hints | the eye-toggle baked into old `Input` |
| `<TextArea>` | auto-grow, counter, multiline | `<Input variant="multiline">` |
| `<NumberInput>` | steppers, locale-aware Intl.NumberFormat, currency, min/max | string concatenation `$${amount}` |
| `<Toggle>` | branded `<Switch>` wrapper, `tone="brand/success/warning/error"` | raw `<Switch>` w/ inline `trackColor` (9 callsites) |
| `<Checkbox>` | single tri-state checkbox (true/false/`'indeterminate'`) | ad-hoc `Pressable` chips |
| `<CheckboxGroup>` | multi-select, vertical/horizontal/columns | the same loop in `FacilityFormScreen.tsx:233-289` |
| `<RadioGroup>` | single-select, horizontal/vertical | `<Tabs variant="segmented">` (semantically wrong) |
| `<Select>` | searchable + optionally-multi · bottom sheet (mobile) / centered popover (web) | admin-only `Select`, mobile had nothing |
| `<Combobox>` | typeahead with optional free-text creation, sync or async | nothing |
| `<DatePicker>` | mode `date`/`time`/`datetime` · platform pickers under one API | free-text `'YYYY-MM-DD'` (`WaiverFormScreen`, `CreateGameScreen`) |
| `<FileUpload>` | drag-and-drop on web, expo pickers on native, normalized `UploadedFile` shape | nothing |

## Standard usage

### Quick standalone

```tsx
import { Input, PasswordInput } from '@sports-yeti/ui';

<Input
  label="Email"
  variant="email"
  value={email}
  onChangeText={setEmail}
  leftAddon="✉︎"
  helpText="We'll send a magic link."
  error={errors.email}
/>

<PasswordInput
  label="Password"
  value={pw}
  onChangeText={setPw}
  showStrength
  isNewPassword
/>
```

### FormField wrapper (recommended)

```tsx
import { FormField, FormRow, Input, Select } from '@sports-yeti/ui';

<FormRow columns={2}>
  <FormField label="City" required>
    <Input value={city} onChangeText={setCity} placeholder="Denver, CO" />
  </FormField>

  <FormField label="Sport">
    <Select options={SPORT_OPTIONS} value={sport} onChange={setSport} />
  </FormField>
</FormRow>
```

### Settings-style row

```tsx
import { FormField, Toggle } from '@sports-yeti/ui';

<FormField
  label="Push notifications"
  description="Game reminders, invites, and replies."
  orientation="row"
>
  <Toggle value={enabled} onValueChange={setEnabled} />
</FormField>
```

### Full RHF + Zod pattern

```tsx
import { z } from 'zod';
import { zodResolver } from '@hookform/resolvers/zod';
import {
  Form,
  FormField,
  FormActions,
  Input,
  PasswordInput,
  useFieldController,
} from '@sports-yeti/ui';

const schema = z.object({
  email: z.string().email('Enter a valid email'),
  password: z.string().min(8, 'At least 8 characters'),
});
type Values = z.infer<typeof schema>;

function EmailField() {
  const ctrl = useFieldController<Values, 'email'>({ name: 'email' });
  return (
    <FormField label="Email" required error={ctrl.error}>
      <Input
        variant="email"
        value={ctrl.value}
        onChangeText={ctrl.onChange}
        onBlur={ctrl.onBlur}
      />
    </FormField>
  );
}

function PasswordField() {
  const ctrl = useFieldController<Values, 'password'>({ name: 'password' });
  return (
    <FormField label="Password" required error={ctrl.error}>
      <PasswordInput
        value={ctrl.value}
        onChangeText={ctrl.onChange}
        onBlur={ctrl.onBlur}
      />
    </FormField>
  );
}

export function LoginScreen() {
  return (
    <Form
      defaultValues={{ email: '', password: '' }}
      resolver={zodResolver(schema)}
      onSubmit={async (values) => login(values)}
    >
      <EmailField />
      <PasswordField />
      <FormActions submitLabel="Sign in" />
    </Form>
  );
}
```

## Accessibility contract (built into every primitive)

- **All controls** announce role + state (`accessibilityRole`, `accessibilityState`) + a `nativeID` linking to the FormField's label and (when present) error/description text.
- **Errors** render with `accessibilityLiveRegion="polite"` so screen readers announce them when validation fires.
- **Disabled state** is communicated via `accessibilityState={{ disabled: true }}` and a 0.7 opacity surface.
- **Focus state** uses a 3px brand-color halo (`box-shadow` on web, native focus ring on iOS) — visible without depending on the border alone.
- **Touch targets** ≥44×44pt on `density='comfortable'` (mobile), ≥32×32px on `'compact'` (admin) per WCAG 2.5.5 / Apple HIG.
- **Color contrast**: every text-on-surface pairing in the new components meets WCAG AA against `surface.bg` and `surface.card`.

## Live reference

Both apps register a `FormControls` route that renders `<FormControlsGallery>` from `@sports-yeti/ui/dev`. Visit:

- **Mobile** — Stack route `FormControls` (e.g., add a temporary nav button: `navigation.navigate('FormControls')`).
- **Admin** — Stack route `FormControls` (route name registered in `MainNavigator.tsx`; you can wire a sidebar entry under "System" by appending to `NAV_GROUPS`).

The gallery exercises every control in three modes: **standalone**, **wrapped in `<FormField>`**, and **driven by `react-hook-form` + `zod`**.

## Migration checklist (next refactor wave)

For every screen you touch:

1. Replace `<TextInput>` → `<Input>` / `<PasswordInput>` / `<TextArea>` / `<NumberInput>`.
2. Replace `<Switch>` → `<Toggle>` inside `<FormField orientation="row">`.
3. Replace ad-hoc `Pressable` chip groups → `<CheckboxGroup>` or `<RadioGroup>`.
4. Replace `<Select>` (admin) and free-text dropdowns → unified `<Select>` from `@sports-yeti/ui`.
5. Replace `'YYYY-MM-DD'` text fields → `<DatePicker>`.
6. Replace `expo-image-picker` callsites + `<input type="file">` → `<FileUpload>`.
7. If the screen has a form, wrap it in `<Form>` with a `zodResolver` and use `useFieldController()` per field.
8. Replace `Alert.alert(...)` errors → inline `<FormField error>` + `<Toast>` for full-form failures.

## Future phases

These were left out of this pass to keep scope tight and the existing screens working:

- **Slider primitive** (price ranges, distance filters).
- **Atomic primitive migration** — moving `Text`/`Card`/`Button`/`Avatar`/`IconBadge`/`Tag`/`Tabs`/`Toast`/`Modal`/`Skeleton`/`EmptyState` from each app's local `ui/` into `@sports-yeti/ui` so the parallel libraries fully merge. Each move requires verifying every screen still renders correctly under the shared theme.
- **ESLint rule** banning raw `TextInput`/`Switch` outside `apps/*/src/ui/` and `Alert.alert` everywhere — currently relies on convention + reviewer enforcement.
- **Storybook (or expanded gallery)** with controls and a11y panels for each component.
- **Dark mode** — the theme provider's `Theme` shape supports it (just add a second token instance and toggle at the provider), but no app currently consumes it.
